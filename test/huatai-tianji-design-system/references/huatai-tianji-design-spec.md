# 华泰天玑设计规范

## 1. 适用范围

本规范用于华泰天玑 Web2.0 体系下的业务工作台、列表页、详情页、表单页、弹窗、抽屉、空状态、分页和图表页面生成。

目标不是制作风格展示页，而是让新页面稳定符合华泰天玑的浅蓝白业务系统气质：

- 页面底层浅蓝到浅青，带柔和径向光斑，整体明亮、干净。
- 主工作区有轻玻璃感，边界柔和，但不喧宾夺主。
- 内容承载层以白色或极浅蓝灰为主，靠浅描边、轻阴影和留白组织层级。
- 交互主色使用蓝色、蓝青色或深蓝文字色。
- 红、绿、橙只用于业务语义，不作为页面主视觉。

## 2. 硬约束

生成页面时必须遵守以下规则：

1. 默认使用浅色页面，不生成深色科技舱、赛博霓虹或重装饰页面。
2. 页面底层必须采用浅蓝到浅青背景，并允许使用柔和径向光斑。
3. 页面结构应优先采用三层视觉：页面底层、玻璃工作区、白色内容承载层。
4. 主按钮、菜单选中、tab 选中、筛选选中、分页选中等交互状态必须使用蓝色体系。
5. 红色仅用于上涨、报错、风险、警示类业务语义；绿色仅用于下跌、安全、成功类业务语义；橙色仅用于提示、预警类业务语义。
6. 内容区应按 1920px 设计基准和 1680px 主面板思路适配，不把内容铺满宽屏。
7. 业务页面必须覆盖默认、hover、active/focus、disabled、error、warning、success 等必要状态。
8. 页面文案只能来自真实业务，不出现“设计说明”“示例数据”“此处替换”“视觉展示”等占位或解释型文案。
9. 空状态必须使用统一浅蓝低饱和风格，不使用强对比插画或深色空状态。
10. 如果原始内容缺失某项信息，应省略或显式展示业务空状态，不补虚构内容。

## 3. 视觉基调

### 3.1 三层结构

页面生成时优先套用以下层级：

1. 页面底层：浅蓝到浅青背景，带柔和径向光斑，整体明亮、干净。
2. 中间工作区：半透明浅蓝玻璃层，浅白边框，轻微 blur，柔和蓝灰阴影。
3. 内容承载层：白色卡片、表格、列表或表单容器，浅蓝灰边框，轻阴影。

推荐 CSS：

```css
.tj-page-shell {
  min-height: 100vh;
  color: #142033;
  background:
    radial-gradient(circle at 8% 12%, rgba(186, 229, 235, 0.72), transparent 30%),
    radial-gradient(circle at 86% 8%, rgba(198, 215, 244, 0.62), transparent 28%),
    linear-gradient(135deg, #eef8fb 0%, #eef4fb 48%, #f4f8fc 100%);

.tj-glass-panel {
  border: 1px solid rgba(255, 255, 255, 0.9);
  border-radius: 20px;
  background: rgba(248, 253, 255, 0.58);
  box-shadow:
    0 30px 80px rgba(99, 143, 179, 0.16),
    inset 0 1px 0 rgba(255, 255, 255, 0.95);

  border: 1px solid rgba(226, 234, 240, 0.96);
  border-radius: 16px;
  background: #ffffff;
  box-shadow: 0 18px 48px rgba(83, 128, 166, 0.1);
}
```

### 3.2 设计倾向

- 明亮、清爽、轻量。
- 信息密度适中，优先保证扫描效率。
- 圆角柔和但不夸张，内容卡片常用 8px、16px，胶囊控件可用 50% 或 999px。
- 阴影要轻，避免厚重浮层。
- 玻璃效果只用于页面工作区、弹窗、抽屉或重点容器，不把每个小卡片都做成玻璃。

## 5. 布局规则

### 5.1 页面基准

- 设计画板：1920x1080。
- 内容可视区域：1680px。
- 主面板：1680x880。
- 左右安全边距：120px。
- 顶部区域高度：80px。
- 底部版权/友情链接区域高度：100px。
- 常见侧导航宽度：120px。
- 侧导航存在时，内容区宽度参考：1440px。

### 5.2 栅格

- 主面板采用 24 栅格体系。
- 列使用 4 的倍数。
- 水槽宽度：24px。
- 页面应适配 1920、1440、1366 和 1280 宽度。

### 5.3 页面组织

推荐结构：

```html
<main class="tj-page-shell">
  <section class="tj-glass-panel">
    <section class="tj-surface">业务内容</section>
  </section>
</main>
```

布局原则：

- 页面外层负责浅蓝到浅青背景和柔和光斑。
- 玻璃工作区负责承接一组业务模块。
- 白色内容层负责表格、表单、图表、详情和列表。
- 不要在卡片里继续套多层卡片；如果必须分组，使用浅描边、分隔线或轻底色。
- 内容密度高时，优先用栅格、表格和分组标题解决，不靠大面积装饰图形。

## 6. 组件规则

### 6.1 按钮

尺寸：

| 类型 | 高度 | 字号 | 用途 |
| --- | --- | --- | --- |
| S | 24px | 14px | 通知提醒框、控件内轻量操作 |
| M | 32px | 16px | 常规弹窗、表格操作栏 |
| L | 40px | 18px | 重要操作、突出按钮 |

规则：

- 按钮文字建议 2-6 个中文字符，最多不超过 8 个中文字符。
- 常规主按钮使用蓝青渐变。
- 次按钮可使用 `#498DF2` 或 `#5CA8E8`。
- 线框按钮使用蓝色描边、蓝色文字、白色或透明背景。
- 图标按钮优先使用线性图标，图标颜色跟随文字色。
- 禁用状态使用低饱和浅蓝，不隐藏按钮。

推荐 CSS：

```css
.tj-button-primary {
  height: 32px;
  border: 0;
  border-radius: 999px;
  padding: 0 18px;
  color: #ffffff;
  background: linear-gradient(90deg, #4486F5 0%, #72C6DD 100%);
}

.tj-button-primary:hover {
  filter: brightness(1.04);
}

.tj-button-primary:disabled {
  color: rgba(255, 255, 255, 0.72);
  background: linear-gradient(90deg, #A9C9F9 0%, #B7D9F5 100%);
  cursor: not-allowed;
}

.tj-button-outline {
  height: 32px;
  border: 1px solid #498DF2;
  border-radius: 999px;
  padding: 0 18px;
  color: #498DF2;
  background: #ffffff;
}
```

### 6.2 图标

PDF 中包含两组线性图标：通用控件图标和业务入口图标。生成页面时应优先使用同一套线性图标语言。

规则：

- 图标采用线性、轻量、单色表达，不混用厚重面性图标、拟物图标或高饱和插画图标。
- 默认图标颜色使用 `#56617C`、`#5D7092` 或同级蓝灰色。
- hover、active、选中状态可使用 `#498DF2`、`#5CA8E8` 或 `#384F97`。
- disabled 状态使用 `#B8C3E5`、`#CEDEEE` 或更浅蓝灰，不直接隐藏图标。
- 图标应与文字色、按钮色、菜单状态联动，避免同一控件内出现多套强调色。
- 工具按钮、搜索、日期、下拉、清空、上传、下载、展开、收起、关闭等控件图标应保持同一视觉尺寸和描边粗细。
- 图标单独作为按钮时必须有明确 hover、active、focus、disabled 状态。
- 业务入口图标可以使用线性轮廓加浅蓝底，但不使用红、橙作为入口主视觉，除非表达风险、报错或预警。

常见图标类型：

| 类型 | 场景 | 颜色建议 |
| --- | --- | --- |
| 控件图标 | 下拉、搜索、日期、关闭、清空、展开、收起 | 默认蓝灰，hover 蓝色 |
| 操作图标 | 新增、编辑、删除、下载、上传、刷新、查看 | 跟随按钮或文字色 |
| 状态图标 | 成功、警告、错误、权限、认证 | 使用对应业务语义色 |
| 业务入口图标 | 行情、资讯、图表、日历、帮助、设置等入口 | 蓝灰线性图标，选中蓝色 |

### 6.3 文本、搜索和日期控件

PDF 中基础表单入口包含文本、下拉、搜索和日期四类控件。

规则：

- 文本输入 placeholder 使用“请输入”。
- 下拉选择 placeholder 使用“请选择”。
- 搜索输入 placeholder 使用“请搜索”，右侧或前置使用线性搜索图标。
- 日期选择器使用线性日历图标，日期范围格式参考 `2020.12.31-2050.12.31`。
- 搜索和日期控件的底色、边框、字号、focus 和 disabled 状态应与输入框规范一致。
- 日期范围过长时不得挤压图标；优先保证文本区域和图标区域稳定。
- 搜索框清空按钮仅在已有输入时显示，颜色使用蓝灰，hover 可转蓝色。
- 日期、搜索、下拉在筛选区组合使用时，控件间距保持一致，不通过颜色制造额外层级。

### 6.4 输入框

规则：

- 最短尺寸：112px。
- 默认背景：`#F6F8FD`。
- 默认边框：`#E1E7EB`。
- focus、输入中：蓝色描边。
- error：红色描边和红色提示。
- warning：橙色描边和橙色提示。
- success：绿色描边和绿色提示。
- 多行文本默认高度：80px，可调整。
- 多行文本右下角显示字数，如 `0/150`。
- 内容超出时允许隐藏，光标滑动后显示。

尺寸与类型：

| 类型 | 场景 | 规则 |
| --- | --- | --- |
| 小尺寸 | 紧凑表单、表格内编辑 | 不小于 112px，保证文字可读 |
| 中尺寸 | 常规表单、筛选区 | 默认推荐尺寸 |
| 大尺寸 | 重要录入、宽字段 | 宽度随字段内容拉长 |
| 带图标输入框 | 搜索、定位、用户、清空等 | 图标置于输入框内，颜色跟随状态 |
| 多行文本 | 备注、说明、长文本录入 | 默认高 80px，右下角显示字数 |
| 数字输入框 | 数量、比例、金额、阈值 | 可带单位或步进控件 |
| 复合型输入框 | 区号+电话、协议+地址、金额+币种、域名后缀 | 前后缀区域用浅底和分隔线区分 |

状态：

| 状态 | 表现 |
| --- | --- |
| 默认 | 浅底、浅蓝灰边框、辅助文字 placeholder |
| hover | 边框略加深，保持轻量 |
| 输入中/focus | 蓝色描边，可加轻微蓝色外发光 |
| 输入完成 | 展示输入内容，文字用正文色 |
| error | 红色边框和红色校验提示 |
| warning | 橙色边框和橙色校验提示 |
| success | 绿色边框和绿色校验提示 |
| disabled 未输入 | 浅灰蓝底、低对比 placeholder |
| disabled 已输入 | 保留内容但降低对比度 |
| 内容超出 | 默认隐藏溢出，光标滑动后显示 |

### 6.5 下拉框与单选选择器

规则：

- 下拉框宽度跟随单选选择器或输入框，可按内容适当拉长。
- 已选中文字：中黑体，14px，`#0D59C4`。
- hover 文字：常规体，14px，`#000000`。
- hover 背景：`rgba(13, 89, 196, 0.08)`。
- 选项高度：40px。
- 输入框有效区域距离上下：8px。
- 超过 10 条出现滚动条。
- 滚动条颜色：`#E1E7EB`。
- 滚动条距上 8px，距右 4px。
- 单项选择器与下拉框组合间距：4px。

状态与结构：

- 单选选择器默认显示“请选择”，已选择状态显示选中值。
- 下拉层可用于内页下拉框，容器使用白底、轻阴影和浅描边。
- 选项数量少时，下拉层高度随内容收缩。
- 选项超过 10 条时出现滚动条，不扩大弹层到影响页面主内容。
- 当前 hover 项使用 `rgba(13, 89, 196, 0.08)` 背景。
- 已选择项文字使用 `#0D59C4`，可配合中黑体强化。
- 下拉框与单项选择器组合时，二者间距为 4px。
- 输入框有效区域距离上下为 8px，避免文字贴边。

### 6.6 标签

规则：

- 默认标签可使用蓝、绿、红、橙、深蓝等体系，但红、绿、橙只表达业务状态。
- 状态标签可使用红、绿、橙，但只表达业务状态。
- 标签文字使用 12px 或 14px。
- 标签可使用胶囊形态。
- 同一行标签过多时，降低色彩差异，避免视觉噪声。

形态：

| 形态 | 用法 | 规则 |
| --- | --- | --- |
| 实心标签 | 需要较强状态识别 | 白色文字，底色使用对应 token |
| 浅底标签 | 常规状态、分类、低强调信息 | 浅色背景，文字使用对应深色 |
| 描边标签 | 筛选项、弱状态、可点击标签 | 白底或透明底，描边和文字同色 |
| 胶囊标签 | 状态、权限、分类 | 圆角使用 50% 或 `999px` |

颜色映射：

| 语义 | 默认色 | 浅底 |
| --- | --- | --- |
| 蓝色/链接入口 | `#4766FF` | `#EBEEFF` |
| 次要蓝 | `#498DF2` | `#E7F0FD` |
| 青蓝 | `#5CA8E8` | `#E4F1FB` |
| 深蓝/主要文字选中 | `#384F97` | `#F0F2F9` |
| 安全/下跌/成功 | `#2DB185` | `#E7F9F3` |
| 上涨/报错/风险 | `#FE4C4F` | `#FFF0F0` |
| 提示/警示 | `#F1913D` | `#FDF2E7` |

### 6.7 表格

规则：

- 表格容器使用白色内容承载层。
- 表头使用极浅蓝灰底或白底加分隔线。
- 表头文字使用 12px 或 14px，中等字重。
- 行 hover 使用浅蓝透明底。
- 当前行、选中行使用浅蓝底和蓝色强调。
- 涨跌、风险、状态等业务语义色可以保留。
- 不使用重网格线，不使用深底反白表格。

### 6.8 分页

规则：

- 支持上一页、下一页、页码、每页条数、跳页输入和总数。
- 当前页使用深蓝或蓝灰底，文字反白。
- 普通页码使用白底、浅描边。
- 每页条数和跳页输入保持输入框规范。
- 总数文案使用辅助文字色。

组合：

| 组合 | 内容 |
| --- | --- |
| 极简分页 | 上一页、当前页、下一页、每页条数、跳页 |
| 标准分页 | 上一页、页码组、下一页、每页条数、跳页 |
| 带总数分页 | 总数、上一页、页码组、下一页、每页条数、跳页 |

状态：

- 上一页、下一页使用线性箭头图标，禁用时降低对比度。
- 当前页使用深蓝或蓝灰底，文字白色。
- 普通页码使用白底、浅描边、正文色。
- hover 页码使用浅蓝底或蓝色描边。
- 每页条数使用下拉控件样式，跳页使用短输入框。
- 总数文案示例为“共 101 项数据”，颜色使用辅助文字色。

### 6.9 空状态

必须覆盖以下常见状态：

- 暂无搜索结果
- 暂无关注
- 暂无数据
- 暂无网络
- 暂无收藏
- 暂无消息
- 流程进行中
- 您还未认证，赶快去认证吧～
- 您还未登录，请登录后查看
- 暂无权限，请进行企业认证
- 即将上线，敬请期待

规则：

- 插画或图标使用浅蓝、蓝灰、低饱和风格。
- 文字使用 `#56617C` 或 `#8D95A5`。
- 空状态不使用强对比红色、橙色或深色背景。
- 空状态可以提供业务下一步入口，但入口按钮必须使用蓝色体系。

### 6.10 风险提示与页脚

- 风险提示使用浅底内容区，文字以正文色或辅助文字色呈现。
- 法务、备案、版权信息使用 12px 或更小字号，颜色使用 `#C1C9D7` 或低对比蓝灰。
- 友情链接和页脚可使用深蓝灰底，但不能抢占主内容注意力。
- 风险提示文案应使用正式法务文本，不临时改写为营销文案。

## 7. 图表规则

- 图表容器使用白色内容承载层。
- 网格线使用极浅蓝灰。
- 主图表颜色优先使用蓝、青、紫、橙等 PDF 图表色，不使用荧光色。
- 如果图表中已有红绿表达涨跌，页面其他模块应减少红绿装饰。
- 图例、坐标、单位使用辅助文字色。
- 图表 hover tooltip 使用白底、浅描边、轻阴影。

## 8. 交互状态

页面生成必须考虑以下状态：

- 按钮：default、hover、active、focus、disabled。
- 图标：default、hover、active、focus、disabled。
- 输入框：default、hover、focus、filled、error、warning、success、disabled、overflow。
- 搜索框：empty、typing、filled、clearable、disabled。
- 日期选择器：default、hover、focus、selected、range selected、disabled。
- 下拉：closed、open、hover option、selected option、scrolling、disabled。
- 标签：default、hover、selected、disabled、semantic status。
- 表格：default row、hover row、selected row、empty。
- tab：default、hover、active、disabled。
- 菜单：default、hover、active、collapsed。
- 分页：default、hover、active、disabled。
- 弹窗/抽屉：打开、关闭、加载中、提交中、错误提示。
- 页面级：loading、empty、error、no permission、not logged in、not certified。

## 10. 输出检查清单

生成前检查：

1. 页面底层是否是浅蓝到浅青背景，并带柔和径向光斑？
2. 是否使用了页面底层、玻璃工作区、白色内容承载层三层结构？
3. 主按钮、tab、菜单、筛选、分页选中是否统一为蓝色体系？
4. 红、绿、橙是否只用于业务语义？
5. 内容区是否符合 1680px 主面板和 24 栅格思路？
6. 字体大小、行高、字重是否来自规范层级？
7. 按钮文案是否控制在 2-6 个中文字符，最多不超过 8 个中文字符？
8. 图标、输入框、搜索、日期、下拉、标签、表格、分页是否覆盖完整交互状态？
9. 空状态是否采用浅蓝低饱和风格？
10. 是否没有出现设计说明、示例数据、占位文案或虚构业务内容？
