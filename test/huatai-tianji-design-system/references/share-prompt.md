# 华泰天玑设计系统启动提示词

把下面提示词连同整个 `huatai-tianji-design-system/` 目录交给协作者或编码助手使用。

```text
你现在需要使用 `huatai-tianji-design-system` skill 设计或改造华泰天玑 Web2.0 页面。

请先读取：

1. `SKILL.md`
2. `references/huatai-tianji-design-spec.md`

执行要求：

- 先判断页面任务、目标用户、核心操作、业务内容和设计介入级别。
- 先做内容盘点、信息层级、低保真布局和组件映射，再进入高保真视觉。
- 高保真视觉必须遵守浅蓝三层结构：页面底层、玻璃工作区、白色内容承载层。
- 主按钮、菜单选中、tab 选中、筛选选中和分页选中统一使用蓝色或蓝青色体系。
- 红、绿、橙只能表达业务语义，不得作为页面主视觉或常规交互主色。
- 页面文案只能来自用户提供内容、已有业务文档或用户确认的信息，不得编造业务数据。
- 不得使用“示例数据”“占位文案”“此处替换”“设计说明”等解释型内容填充高保真页面。
- 未经用户明确允许，不引用、复用或改造现成图片、截图、素材图、历史 HTML 页面或模板文件。
- 必须补全按钮、输入、下拉、表格、tab、分页、弹窗、抽屉和页面级状态。

如果用户要求生成或修改页面代码，请先给出简短方案，说明改造范围、是否涉及现成素材或历史文件、风险和验证方式，等待确认后再实现。

输出时至少包含：

- 页面任务定义
- 设计介入级别
- 内容盘点
- 体验优化机会点
- 信息层级表
- 布局方案
- 新增或调整模块清单
- 内容到组件映射
- 高保真规范应用说明
- 状态清单
- 风险与待确认项
```

## 分享检查

- 发送整个 `huatai-tianji-design-system/` 目录。
- 确认 `references/huatai-tianji-design-spec.md` 能被协作者打开。
- 如果目标项目缺少字体、图标或图表库，按项目现有体系做等价映射，不强行新增依赖。
- 如果需要同时发送 PDF、截图或历史页面文件，先确认授权和使用范围。
