# 华泰天玑全系统页面模式 v2

## 0. 使用说明

本文件是“全系统页面改版方法”，主要来自原 `huatai-tianji-design-system` 的落地经验和 v2 对真实前端全系统样式改版的补充。

执行时必须注意：

- 视觉 token、字体、阴影、圆角、按钮、输入框、下拉框、标签等硬规则，以同目录 `./huatai-tianji-design-spec.md` 中的 V2 内置硬规范为准。
- 本文件用于判断页面类型、组织信息层级、选择改版重点。
- 如果本文件与 V2 内置硬规范冲突，以 V2 内置硬规范和用户本次指定系统代码、目标页面、截图标注及修改范围为准。
- 如果用户要求“其它不要动”，只修改用户指定区域，不套用全页模式。
- 页面模式不是重排真实代码布局的许可。真实前端代码改版时，默认保留原页面结构、边距、对齐、按钮尺寸、弹窗尺寸、图片尺寸和表格列宽，优先做颜色层和状态层改版。
- 本文件中的尺寸建议主要用于新建页面、原页面明显不统一、原页面已经破版或用户明确要求统一布局时参考。

## 1. 页面分类

天玑前端页面改版前，先判断页面属于哪一类。不同页面不能套同一种卡片布局。

| 页面类型 | 典型内容 | 改版重点 |
| --- | --- | --- |
| 门户/首页 | 顶部导航、业务入口、公告、资讯、风险提示 | 第一屏品牌和业务入口清晰，避免营销化 |
| 工作台/看板 | 指标、待办、快捷入口、图表、列表 | 信息密度、核心任务、状态优先 |
| 列表/表格页 | 筛选、批量操作、表格、分页 | 查询效率、列宽稳定、操作列清晰 |
| 详情页 | 字段、状态、附件、流程、记录 | 固定字段行结构，层级稳定 |
| 表单页 | 分组字段、校验、上传、提交 | 标签和值对齐，校验状态完整 |
| 配置页 | 开关、选项、规则、保存 | 控件状态、分组说明、风险提示 |
| 图表分析页 | 指标、趋势图、筛选、图例、说明 | 图表可读性、单位、tooltip、数据状态 |
| 认证/开户流程 | 步骤条、表单、上传、确认、结果 | 步骤清晰、主操作明确、错误可恢复 |
| 消息/通知页 | 分类、列表、未读、详情跳转 | 未读状态、时间、操作入口 |
| 协议/法务页 | 长文、表格、风险说明、操作 | 阅读稳定、文字密度、操作克制 |
| 弹窗/抽屉 | 表单、确认、详情、筛选 | 尺寸、标题、底部操作、滚动 |
| 空/异常/权限页 | 空数据、错误、未登录、未认证 | 真实状态，不编造数据 |

## 2. 门户/首页

结构建议：

```text
顶部导航
第一屏主区域
  品牌/业务信号
  核心入口
  重要公告或风险提示
内容区
  业务模块
  资讯/研究/服务入口
页脚
```

规则：

- 第一屏必须能看出是华泰天玑或华泰期货业务系统。
- 不做大幅营销 hero，不使用夸张插画。
- 导航和用户区保持克制。
- 业务入口使用线性图标和浅蓝底，不用高饱和装饰。

## 3. 工作台/看板

结构建议：

```text
页面标题 + 更新时间 + 主操作
核心指标区
待办/预警/快捷入口
图表或趋势区
明细列表
```

规则：

- 首屏优先展示用户要处理的任务。
- 指标卡不超过 4-6 个。
- 待办和预警用语义色，但面积小。
- 图表和表格不要同时过度抢占视觉。
- 空数据展示真实空状态。

## 4. 列表/表格页

结构建议：

```text
页面标题
筛选区
  搜索
  下拉
  日期范围
  查询/重置
操作区
  新增/导出/批量操作
表格
分页
```

尺寸建议：

- 筛选控件高度 32-40px。
- 表头高度 48-52px。
- 表格行高 52-56px。
- 操作列宽度 120-220px。
- 分页右下或底部居中，保持稳定。

规则：

- 筛选区过多时分为基础筛选和高级筛选。
- 表格列不要随内容抖动。
- 操作列使用文字按钮，危险操作红色。
- 批量操作必须有选中态和禁用态。
- 空搜索结果和暂无数据要区分。

## 5. 详情页

结构建议：

```text
页面标题 + 状态 + 返回/编辑
基础信息
业务信息
附件/材料
流程/记录
风险或说明
```

字段行规则：

- 字段名宽度固定：120-160px。
- 值区域自适应。
- 操作列固定：120-200px。
- 普通行高 48-56px。
- 头像、二维码、上传材料行高 96-112px。

规则：

- 不用自由 `gap` 混排普通行和媒体行。
- 分组标题 16-18px。
- 状态靠近标题或关键字段。
- 风险说明放在底部或相关字段下方。

## 6. 表单页

结构建议：

```text
页面标题/说明
表单分组
  字段标签
  输入控件
  辅助说明/校验提示
附件上传
风险说明
底部操作
```

规则：

- 标签和值对齐。
- 必填项标识清晰但不喧宾夺主。
- 控件高度统一。
- 上传区高度稳定。
- 禁用字段内容仍可读。
- 底部操作固定或靠近表单结束位置。

## 7. 配置页

适用于偏好设置、通知设置、权限设置、规则配置。

结构建议：

```text
页面标题
配置分组
  配置项名称
  当前状态/说明
  控件
保存操作
```

规则：

- 开关、单选、多选、标签选择要有明确状态。
- 配置说明使用辅助文字，不做大段解释。
- 危险配置需要二次确认。
- 保存按钮禁用态要明确。

## 8. 图表分析页

结构建议：

```text
标题 + 数据更新时间
筛选区
核心指标
主图表
辅助图表/明细表格
数据说明
```

规则：

- 图表容器白底。
- 坐标轴、图例、单位低噪声。
- tooltip 白底轻阴影。
- 红绿仅用于涨跌等业务语义。
- 图表为空时显示图表空状态，不伪造数据。

## 9. 认证/流程页

结构建议：

```text
步骤条
当前步骤说明
表单/上传/扫码
上一步/下一步/保存
风险说明
```

规则：

- 当前步骤蓝色。
- 已完成步骤绿色或蓝色勾选均可，但绿色只表达完成。
- 未完成步骤浅灰蓝。
- 步骤过多时横向均分或折叠。
- 错误状态必须可恢复。

## 10. 弹窗

常见类型：

- 确认弹窗。
- 表单弹窗。
- 详情弹窗。
- 风险确认弹窗。

尺寸：

- 小确认：420-520px。
- 常规表单：640-800px。
- 复杂表单：800-960px 或改用抽屉。

规则：

- 标题、关闭、内容、底部操作结构固定。
- 表单字段间距稳定。
- 底部按钮顺序保持项目习惯。
- 高风险操作红色按钮或红色文本。

## 11. 抽屉

适用于：

- 明细查看。
- 复杂筛选。
- 编辑配置。
- 操作记录。

规则：

- 宽度 480/560/720px。
- 标题固定。
- 内容滚动。
- 底部操作固定。
- 不在抽屉内再嵌套大弹窗。

## 12. 空状态/异常页

状态类型：

| 状态 | 文案方向 | 操作 |
| --- | --- | --- |
| 暂无数据 | 当前范围暂无数据 | 可刷新或新增 |
| 暂无搜索结果 | 调整筛选条件 | 重置筛选 |
| 暂无权限 | 当前账号无权限 | 联系管理员 |
| 未登录 | 登录后查看 | 去登录 |
| 未认证 | 完成认证后使用 | 去认证 |
| 网络异常 | 网络或服务异常 | 重试 |
| 加载失败 | 数据加载失败 | 重新加载 |

规则：

- 不用“示例数据”填充。
- 不用夸张插画。
- 主操作使用蓝色体系。
