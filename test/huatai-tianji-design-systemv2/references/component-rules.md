# 华泰天玑组件细则 v2

## 0. 使用优先级

组件改版时按以下顺序判断：

1. 用户本次指定的系统代码、目标页面、组件范围、截图、标注和修改要求。
2. 本 V2 文件夹已内置的组件硬规则。
3. 同目录 `./huatai-tianji-design-spec.md` 中的核心视觉 token。
4. 本文件中的全系统落地补充规则。

标注为“V2 内置硬规范”的规则优先执行；标注为“落地补充”的规则用于真实代码改版时统一细节。

## 1. 组件通用规则

- 同类组件高度、圆角、颜色、字重必须统一。
- 组件状态至少覆盖 default、hover、active、focus、disabled。
- 业务语义状态补充 error、warning、success。
- 图标优先线性风格，尺寸 14-20px。
- 不用难看的文本符号替代图标。
- 组件文本不得溢出、遮挡或重叠。
- 红、绿、橙只用于业务语义，不用于普通选中态。
- 普通筛选项、商品 chip、合约选择、分段选择器、Tab、菜单、分页、快捷筛选的选中态不得使用红色、粉红或浅红背景。
- 未读消息徽标允许红色小面积提示；品牌 logo 允许保留品牌红；图表橙色线作为数据序列色保留。
- 真实前端代码改版时，组件尺寸规则默认作为参考，不主动覆盖原有可用尺寸；只有新建组件、明显不统一、原页面破版或用户明确要求统一时才调整高度、宽度、padding、行高和布局对齐。
- 真实前端代码改版优先改颜色、背景、边框、分割线、文字色、图标色、hover/active/focus/disabled 状态。

## 2. 按钮（V2 内置硬规范）

尺寸使用原则：

- V2 L/M/S 尺寸用于新建组件、明显不统一、原页面破版或用户明确要求统一尺寸时参考。
- 真实前端代码改版时，默认不主动修改已有按钮高度、宽度和 padding。
- 优先改按钮颜色、背景、描边、文字色、hover、disabled 等视觉状态。
- 按钮改版前必须先全量扫描目标页面所有按钮，不只看 `.primary`、组件库主按钮或当前可见的第一个按钮。

按钮全量覆盖审查：

- 扫描主按钮、次按钮、线框按钮、文字按钮、图标按钮、表格操作按钮、弹窗底部按钮、固定底部按钮、自定义按钮。
- 扫描常见业务按钮：分享、导出、导入、上传、下载、刷新、查询、重置、取消、保存、提交、确认、删除、撤回。
- 每个按钮都要判断业务语义，再决定颜色：普通操作用蓝色或蓝青体系，风险操作才用红色，成功/完成才用绿色，警示/待处理才用橙色。
- “分享”通常属于普通次级操作，默认使用蓝色文字/蓝色描边/浅蓝 hover，不应使用红色，除非业务明确表示风险。
- 需要同时检查 default、hover、active、focus、disabled 状态。
- 不为了统一视觉主动改按钮高度、宽度、padding、display 或对齐方式；只有破版或用户明确要求时才改。

### 2.1 按钮尺寸

| 尺寸 | 高度 | 字体 | 适用场景 |
| --- | --- | --- | --- |
| L | 40px | 18px | 重要操作、关键确认、强调型操作 |
| M | 32px | 16px | 常规弹窗、表格操作栏、常规操作 |
| S | 24px | 14px | 轻量按钮、紧凑控件内操作 |

按钮文字：

- 建议不超过 8 个中文字符。
- 推荐 2-6 个中文字符。
- 长按钮文案应优先缩短业务文案；不能缩短时允许增加水平 padding，不压缩字号到不可读。

### 2.2 基础按钮

用途：

- 查询、保存、提交、确认、下一步。
- 普通主操作优先使用蓝色或蓝青色体系。
- 风险确认才允许使用红色体系。

样式：

- 新建、明显不统一、原页面破版或用户明确要求统一时，高度按 L/M/S 使用 40/32/24px。
- 背景使用 V2 主色或次按钮色板。
- 普通主按钮优先 `#4486F5` 或 `#4486F5` 到 `#72C6DD`。
- 白色文字。
- 禁用态使用低饱和浅色，不让按钮完全消失。

### 2.3 线框按钮

用途：

- 编辑、修改、取消、上一步、重置、次级操作。

样式：

- 白底或极浅蓝底。
- 边框使用蓝色体系。
- 文本使用蓝色体系。
- hover 可使用浅蓝背景。
- disabled 使用低对比蓝灰。

### 2.4 带图标按钮

用途：

- 上传、下载、搜索、刷新、导出、关闭、展开。

样式：

- 图标使用线性风格。
- 图标与文字间距 4-8px。
- 图标颜色跟随按钮文字色。
- 不用文本箭头、文本叉号、文本加号替代图标。

### 2.5 风险按钮

用途：

- 删除、撤回、注销、拒绝、失败确认、高风险提交。

样式：

- 使用红色语义：`#FE4C4F`、`#FE8082`、`#FFB3B4`、`#FFF0F0`。
- 根据风险强度使用红色描边、红色文字或红色实心。
- 不用于普通确认。

## 3. 输入框（V2 内置硬规范）

结构：

```text
label
input/select/date/search
help/error text
```

规则：

- 高度 40px。
- 圆角优先 8px。
- 背景 `#F6F8FD`。
- 边框 `#E1E7EB`。
- 文本 14px。
- placeholder 使用 `#8D95A5` 或更浅辅助色。
- focus 使用蓝色描边。
- error 使用红色描边和红色提示。
- warning 使用橙色描边和橙色提示。
- success 使用绿色描边和绿色提示。
- disabled 使用浅灰蓝背景，内容仍可读。

## 4. 下拉框（V2 内置硬规范）

### 4.1 选择器

- 选择器输入态高度 40px。
- 默认文字使用 14px。
- 已选中文字：中黑体，14px，颜色 `#0D59C4`。
- 默认箭头使用线性 chevron，不拼接文本符号。
- 单选选择器与下拉框组合时，间距 4px。

### 4.2 下拉层

- 白底，浅描边，轻阴影。
- 选项高度 40px。
- hover 文字：常规体，14px，颜色 `#000000`。
- hover 背景：`#0D59C4`，8% 透明度。
- 输入框有效区域距离输入框上下为 8px。
- 超过 10 条出现滚动条。
- 滚动条颜色 `#E1E7EB`。
- 滚动条距离上方 8px，距离右侧 4px。
- 下拉层不得遮挡关键操作区；如果会遮挡，优先调整弹出方向或容器滚动。

## 5. 搜索、日期与筛选（V2 内置硬规范 + 落地补充）

搜索框：

- 高度 40px。
- 可带搜索图标。
- 搜索图标使用线性风格。
- 有输入时可显示清除图标。
- placeholder 使用“请输入关键词”或项目已有文案。

日期框：

- 与输入框保持 40px 高度。
- 日期图标使用线性风格。
- 禁止用文本符号代替日期图标。

筛选区：

- 控件高度统一。
- 查询按钮靠近筛选项。
- 重置使用次按钮或文本按钮。
- 高级筛选折叠时保留核心条件。
- 筛选项过多时换行，而不是压缩到不可读。

筛选标签 / 商品 chip / 合约选择 / 分段选择器：

- 普通筛选类控件属于信息选择，不属于风险语义。
- default：白底或极浅蓝灰底，浅蓝灰边框，正文色文字。
- hover：浅蓝底，蓝色边框或蓝色文字。
- active/selected：使用浅蓝底 `#E7F0FD` 或 `rgba(68, 134, 245, 0.10)`，边框使用 `#4486F5`，文字使用 `#0D59C4` 或 `#4486F5`。
- active/selected 不得使用红色、粉红、浅红背景或红色描边。
- disabled 使用低对比蓝灰，仍保持可读。
- 红色只允许用于风险、失败、报错、上涨、高风险确认等明确业务语义，不用于当前筛选项。

## 6. 标签（V2 内置硬规范）

标签状态来自 V2 内置色板，可使用实底、浅底、描边等形式。

| 类型 | 色系 | 用途 |
| --- | --- | --- |
| 分类标签 | 蓝色、蓝灰、浅底 | 普通分类、筛选结果、信息标记 |
| 成功/安全 | `#2DB185` 体系 | 成功、安全、下跌、已完成 |
| 风险/失败 | `#FE4C4F` 体系 | 风险、失败、报错、上涨 |
| 警示/待处理 | `#F1913D` 体系 | 提示、警示、待处理 |
| 链接/入口 | `#4766FF` 体系 | 可点击入口、跳转 |

规则：

- 文字 12-14px。
- 圆角可用 999px、8px 或 4px，保持同页面统一。
- 同一行标签数量多时降低颜色强度。
- 状态标签颜色必须对应业务语义。
- 不使用红、绿、橙表达普通选中。

## 7. 图标（V2 内置硬规范）

- 使用统一线性图标风格。
- 常规尺寸 14-20px。
- 常规颜色使用 `#56617C`、`#8D95A5` 或蓝色体系。
- 危险、成功、警示图标可使用对应语义色。
- 图标按钮 hover 需要有轻微反馈。
- 不用过小的文本符号替代关闭、返回、展开、搜索、日期、上传、下载等图标。

## 8. 表格（落地补充）

表头：

- 极浅蓝灰底。
- 文字 14px/20px，字重 700。
- 高度 48-52px。

表体：

- 行高 52-56px。
- 分割线 `#E1E7EB`。
- hover 使用浅蓝透明底。
- selected 行使用浅蓝底。

操作列：

- 宽度固定。
- 操作按钮横向排列，间距 12-16px。
- 查看/编辑使用蓝色。
- 删除/撤回使用红色。

空表格：

- 保留表头。
- 表体展示空状态。
- 暂无数据和暂无搜索结果要区分。

## 9. 分页（落地补充）

- 支持上一页、下一页、页码、每页条数、跳页、总数。
- 当前页使用蓝色底或深蓝文字。
- 禁用态降低对比，但保留位置。
- 分页位置稳定，通常右下。
- 不因页码变化导致表格主体跳动。

## 10. Tab（落地补充）

- 选中使用蓝色体系。
- 可采用下划线或胶囊样式，但同页面统一。
- hover 有轻微反馈。
- disabled 低对比。
- 不使用红、绿、橙做选中。
- 商品、合约、交易所等业务 Tab 或分段选择器同样按蓝色/蓝青选中态处理，不使用红色、粉红或浅红背景。

## 11. 菜单（落地补充）

顶部导航：

- 文字优先 14px。
- 当前项蓝色文字 + 蓝青下划线。
- hover 轻微变蓝。

侧边导航：

- 一级选中使用蓝色或蓝青体系，文字清晰。
- 子项选中浅蓝底，蓝字。
- 展开图标使用 chevron。
- 如项目已有“同一时刻最多展开一个一级分组”的交互逻辑，视觉上只高亮当前点击项；不要因为样式改版改变菜单展开/收起逻辑。
- 多个菜单项指向同一页面时，只高亮当前点击项。

## 12. 卡片与内容容器（落地补充）

用途：

- 指标卡。
- 业务入口卡。
- 企业认证卡。
- 通知卡。
- 管理入口卡。

规则：

- 圆角 8px 或 16px。
- 白底或极浅蓝灰。
- 阴影优先参考 V2 内置硬规范：`0 0 30px rgba(145, 152, 167, 0.25)`，真实页面可降低透明度。
- 不做卡片套卡片。
- 卡片之间间距 16-24px。

## 13. 弹窗（落地补充）

结构：

```text
mask
modal
  header: title + close
  body
  footer: actions
```

规则：

- mask 使用半透明深蓝黑。
- modal 白底。
- 标题 18px/24px。
- close 使用图标。
- footer 按钮统一。
- 内容过长时 body 滚动。
- 高风险操作使用红色按钮或红色文本。
- 真实代码改版时默认保留原弹窗宽高、位置和内容布局；优先调整背景、描边、阴影、标题色、按钮色和状态。
- 只有原弹窗明显破版、尺寸不统一影响使用，或用户明确要求时，才调整弹窗尺寸。

## 14. 开关、单选、多选（落地补充）

开关：

- 开启使用蓝色或蓝青体系。
- 关闭使用浅灰蓝。
- 禁用降低对比。

单选/多选：

- 选中使用蓝色。
- 错误提示使用红色。
- 选项间距稳定。
- 不用绿色表达普通选中。

## 15. 上传（落地补充）

- 上传框浅底、虚线或浅描边。
- 支持默认、hover、上传中、成功、失败、禁用状态。
- 已上传文件展示文件名、大小、删除/预览操作。
- 上传图标使用线性风格。
- 失败状态使用红色提示。

## 16. 步骤条（落地补充）

- 当前步骤使用蓝色。
- 已完成步骤使用勾选状态。
- 未完成步骤使用浅灰蓝。
- 步骤文字 13-14px。
- 横向空间不足时压缩文字或改竖向，不重叠。

## 17. 图表容器（V2 内置硬规范 + 落地补充）

- 图表色使用同目录 `./huatai-tianji-design-spec.md` 中的 V2 内置图表色。
- 容器使用白底 surface。
- 标题区可包含标题、单位、更新时间或筛选。
- 图例低噪声。
- 坐标轴、图例、单位使用辅助文字色。
- tooltip 白底、浅描边、轻阴影。
- 空状态不伪造数据。
- 如果红绿已用于涨跌，其它装饰区域应减少红绿使用。
