# 车友会微信小程序原型设计规范

适用范围：车友会微信小程序用户端、合作方端、商家端、小程序内管理者工作台的 Pencil 原型设计。

参考文档：《车友会微信小程序完整 PRD v0.52》

## 1. 设计定位

产品气质：真实车主社群、城市车友会、活动运营、车主服务和轻商业化平台。

设计关键词：

- 可信：认证车主、分会、活动、订单、权益都要有清晰状态。
- 高效：用户能快速完成定位、选择分会、认证、报名、下单、查看消息。
- 社群感：头像、车型、分会、兴趣标签、活动照片要让用户感到有人气。
- 运营友好：活动、积分、优惠券、商品、推荐位和审核管理要方便后续配置。
- 小程序优先：界面按微信小程序体验设计，不做 App 式复杂沉浸导航。

## 2. 原型尺寸与画板

| 类型 | 建议尺寸 | 用途 |
| --- | --- | --- |
| 微信小程序标准页面 | 375 x 812 | iPhone 主画板，用于绝大多数用户端页面 |
| 长页面 | 375 x 1200/1600 | 活动详情、商品详情、分会首页、认证表单等 |
| 弹窗/半屏抽屉 | 375 x 812 | 在同画板中标注遮罩、底部弹层和确认弹窗 |
| 管理者工作台页面 | 375 x 812 或 390 x 844 | 小程序内管理页面，不做 Web 后台宽屏 |
| 流程总览图 | 1440 x 900 | 用于展示完整流程、状态流转和页面关系 |

画板命名：

```text
流程编号_页面序号_页面名称_状态
示例：03_01_车主信息填写_默认
示例：07_04_活动报名确认_收费活动
```

## 3. 栅格与间距

| 项目 | 规范 |
| --- | --- |
| 页面左右边距 | 16 px |
| 卡片内边距 | 12-16 px |
| 模块上下间距 | 16-24 px |
| 表单字段间距 | 12-16 px |
| 按钮高度 | 44-48 px |
| 底部固定操作区 | 高度 72-88 px，需避开安全区 |
| 圆角 | 普通卡片 8 px；按钮 8 px；标签 4-6 px；头像圆形 |

原则：

- 不要卡片套卡片。
- 同一页面内同级模块间距保持一致。
- 列表页要优先保证信息密度，避免过大留白。
- 底部主操作按钮固定时，页面内容底部需预留空间。

## 4. 色彩规范

| 用途 | 颜色建议 | 说明 |
| --- | --- | --- |
| 主色 | #FFB600 | 用于主按钮、重点状态、品牌强调 |
| 辅助蓝 | #1677FF | 用于链接、导航、地图、信息提示 |
| 成功绿 | #22A06B | 认证通过、报名成功、已签到、已领取 |
| 警示橙 | #FA8C16 | 候补、待审核、即将开始、待支付 |
| 危险红 | #E5484D | 驳回、取消、失败、违规、删除 |
| 金标色 | #C6922E | 会长、圈主等管理审核身份 |
| 银标色 | #8A94A6 | 认证车型银标 |
| 页面背景 | #F6F7F9 | 小程序通用浅灰背景 |
| 卡片背景 | #FFFFFF | 信息卡片、表单卡片 |
| 主文字 | #111827 | 标题、正文重点 |
| 次文字 | #6B7280 | 描述、说明 |
| 弱文字 | #9CA3AF | 时间、补充信息、禁用态 |
| 分割线 | #E5E7EB | 列表分割和边框 |

使用原则：

- 主色只用于关键行动，不要满屏黄色。
- 金色只用于会长/圈主金标和少量荣誉标识。
- 商业推广内容必须用“合作推荐”“赞助支持”“车主福利”等标签标识。
- 错误、取消、删除等高风险动作统一用红色。

## 5. 字体与排版

| 层级 | 字号 | 字重 | 用途 |
| --- | --- | --- | --- |
| 页面标题 | 20-24 | 600/700 | 首页模块、详情页标题 |
| 模块标题 | 17-18 | 600 | 卡片标题、分区标题 |
| 正文 | 14-16 | 400 | 列表正文、表单内容 |
| 辅助文字 | 12-13 | 400 | 时间、说明、状态补充 |
| 标签文字 | 10-12 | 500 | 状态标签、车型银标、兴趣标签 |
| 按钮文字 | 15-16 | 600 | 主按钮、底部操作按钮 |

排版规则：

- 页面内标题不要过大，避免像营销落地页。
- 列表卡片标题最多 2 行，超出省略。
- 活动、商品、帖子详情正文可多行展示，但首屏需露出核心操作。
- 表单说明文字要短，风险说明可放在字段下方或提交按钮上方。

## 6. 图标与视觉元素

图标建议使用线性图标风格，保持 2 px 线宽。

常用图标：

| 场景 | 图标建议 |
| --- | --- |
| 首页 | home、map-pin、calendar、message、user |
| 活动 | calendar、clock、map-pin、users、ticket、check-circle |
| 社区 | image、heart、message-circle、share、bookmark |
| 商城 | shopping-bag、package、truck、credit-card |
| 积分 | gift、star、trophy、ticket-percent |
| 管理 | shield-check、settings、list-check、bar-chart |
| 风控/警示 | alert-triangle、x-circle、lock |

视觉资产：

- 活动、分会、商品、专题、婚车招募等页面需要封面图占位。
- 没有真实图片时，用灰色图片占位框 + 图片说明，不用抽象插画堆砌。
- 往期活动照片、车友动态、商品图、门店图是核心视觉内容，原型中需留出真实媒体位置。

## 7. 导航规范

底部导航首版建议：

| Tab | 说明 |
| --- | --- |
| 首页 | 分会、推荐活动、推荐内容、权益提醒 |
| 社区 | 我的/广场、车友圈、内容流 |
| 活动 | 活动列表、活动状态、报名入口 |
| 车主服务 | 贴膜、保养、洗美、保险、购车换车等服务 |
| 我的 | 资料、车辆、积分、订单、消息、管理者入口 |

页面顶部：

- 小程序顶部遵循微信胶囊区域，不在原型里遮挡右上角区域。
- 二级页面使用返回箭头 + 页面标题。
- 搜索、筛选、更多操作放右侧或标题下方工具栏。

## 8. 通用组件

### 8.1 按钮

| 类型 | 用途 |
| --- | --- |
| 主按钮 | 登录、提交认证、立即报名、立即支付、确认领取 |
| 次按钮 | 查看详情、暂不登录、稍后完善、联系小助手 |
| 文本按钮 | 编辑、删除、取消、查看规则 |
| 危险按钮 | 取消活动、删除内容、强制退报名、注销成员关系 |
| 固定底部按钮 | 报名、支付、提交、保存等关键动作 |

按钮状态：

- 默认
- 按下
- 禁用
- 加载中
- 已完成
- 风险确认中

### 8.2 卡片

常用卡片：

- 分会卡片：城市、分会名、成员数、活动数、申请状态。
- 活动卡片：封面、标题、状态、时间、位置、费用、报名人数。
- 帖子卡片：头像、昵称、车型/身份标、内容摘要、图片、点赞评论。
- 商品卡片：图片、名称、价格、积分抵扣、合作推荐标。
- 服务卡片：服务名、门店、价格/优惠、小助手入口。
- 消息卡片：消息类型、标题、摘要、时间、未读状态。
- 管理卡片：待处理数量、模块入口、风险状态。

### 8.3 标签

状态标签：

- 待审核
- 已通过
- 已驳回
- 招募中
- 已满员
- 候补中
- 待支付
- 已报名
- 已签到
- 已结束
- 已取消

身份标签：

- 会长金标
- XX 圈主金标
- 认证车型银标
- 认证车主徽章
- 合作推荐
- 赞助支持

## 9. 表单规范

表单结构：

```text
页面标题
说明文案
表单分组
字段
字段说明/示例
风险或隐私提示
底部提交按钮
```

字段状态：

- 默认
- 已填写
- 必填未填
- 格式错误
- 审核中
- 已锁定
- 已自动拉取

关键表单：

- 登录注册：头像、昵称、手机号。
- 车主认证：米 ID、车牌、车型、配置、颜色、小米汽车 App 截图。
- 活动报名：报名人信息、车辆信息、同行人备注、费用确认。
- 合作方资料：合作类型、城市、资源、诉求、联系方式。
- 我要找婚车：时间、车型、颜色、数量、地点、金额、备注、联系方式。
- 商城订单：SKU、收货地址、积分抵扣、支付金额。
- 管理者发布活动：时间、报名截止、位置名称、详细场地说明、导航地址、报名配置。

## 10. 权限与拦截规范

游客拦截：

```text
游客模式不可操作，请先登录
```

车主专属：

```text
该功能仅限认证车主使用
```

合作方限制：

```text
该功能仅限车主使用
```

分会限制：

```text
该活动仅限本分会成员报名
```

抓车友限制：

```text
请先完成车主认证
```

设计要求：

- Toast 用于轻提示。
- 权限阻断页用于强引导，如去登录、去认证。
- 高风险操作必须用二次确认弹窗。
- 禁用按钮需要说明原因，不能只置灰。

## 11. 状态页规范

每个核心流程至少要覆盖以下状态：

| 状态 | 示例 |
| --- | --- |
| 空状态 | 暂无活动、暂无帖子、暂无订单、暂无消息 |
| 加载中 | 骨架屏或加载占位 |
| 成功 | 认证提交成功、报名成功、支付成功、领取成功 |
| 失败 | 支付失败、提交失败、网络异常 |
| 待审核 | 认证待审核、合作方待联系、婚车需求待审核 |
| 驳回 | 认证驳回、入会申请驳回、内容审核驳回 |
| 权限不足 | 游客、合作方、未认证用户、非本分会成员 |
| 风控限制 | 积分异常、抓车友限频、内容违规 |

## 12. 消息与通知规范

消息中心分类：

- 全部
- 活动消息
- 系统消息
- 互动消息

消息卡片需包含：

- 消息类型
- 标题
- 摘要
- 时间
- 未读红点
- 跳转目标

活动报名成功消息需展示：

- 活动名称
- 活动时间
- 活动位置名称
- 详细场地说明
- 导航地址
- 注意事项
- 详情跳转
- 地图导航入口

## 13. 地址与地图规范

线下活动地址展示采用三段式：

| 字段 | 是否用户可见 | 示例 |
| --- | --- | --- |
| 活动位置名称 | 是 | 深圳湾网球中心 |
| 详细场地说明 | 是 | 3 楼 5 号场，入口右转上楼 |
| 导航地址 | 是 | 深圳市南山区某某路 88 号 |
| 地图导航位置 | 点击导航使用 | 地图 POI/经纬度 |
| 内部地址备注 | 否 | 管理者内部组织备注 |

普通用户端要能看清楚具体集合点或场地信息，不能只展示模糊地点。

## 14. 积分与权益设计规范

积分模式：

```text
后台可配置规则
  + 每日总上限
  + 单任务上限
  + 风控撤销
```

原型表现：

- 积分中心展示当前积分、连续签到、今日任务、可兑换内容。
- 任务卡片展示奖励积分、完成状态、领取按钮和上限提示。
- 达到上限时提示“今日积分已达上限”或“该任务今日已达上限”。
- 积分明细展示获得、消耗、撤销、扣回。
- 管理者工作台展示积分规则配置、流水审核、异常扣回。

## 15. 管理者工作台设计规范

管理端仍是小程序页面，不做 Web 后台风格。

设计原则：

- 首页先展示待处理事项和关键数据。
- 列表页支持筛选、搜索、状态标签和批量感知。
- 详情页突出审核资料、风险信息和操作记录。
- 高风险操作必须二次确认并要求填写原因。
- 本地会长/管理员只看授权分会；平台运营可看全局。

管理常用页面：

- 待审核列表
- 用户详情
- 认证审核详情
- 活动管理
- 报名名单
- 内容审核
- 积分流水
- 订单详情
- 合作方线索
- 商家核销
- 数据看板

## 16. 文件命名规范

Pencil 文件：

```text
流程编号_流程名称_原型_v0.1.pen
示例：07_活动浏览报名支付_原型_v0.1.pen
```

页面说明：

```text
流程编号_流程名称_页面说明.md
```

导出目录：

```text
exports/
```

版本建议：

- `v0.1`：低保真结构稿
- `v0.2`：补齐状态和异常
- `v0.3`：接近评审稿
- `v1.0`：确认稿

## 17. 原型交付检查清单

每个流程原型完成后需检查：

- 是否覆盖主流程。
- 是否覆盖游客、注册用户、认证车主、合作方、管理者等关键权限。
- 是否覆盖空状态、成功、失败、待审核、驳回、禁用态。
- 是否有返回路径。
- 是否有消息触达或结果反馈。
- 是否标注后台配置项。
- 是否标注微信支付、订阅消息、地图导航、小助手等外部能力。
- 是否避免展示手机号、完整车牌、微信号、米 ID、认证截图等敏感信息。
- 是否符合本设计规范的颜色、间距、按钮和标签规则。
