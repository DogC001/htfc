# DH Web HTML 原型设计规范

适用范围：基于旧版页面截图、现有 HTML、UI 设计规范和用户改版要求，生成可编辑的 Web HTML 高保真原型。

参考沉淀：

- `AGENTS.md`
- `DH-skills.md`
- `huatai-tianji-design-system/references/huatai-tianji-design-spec.md`
- 本项目中 `my-account-v2.html`、`my-account-v3.html` 的迭代经验

## 1. 设计定位

产品气质：清爽、可信、高效、克制的业务系统界面。

设计关键词：

- 稳定：保留原页面布局、信息结构和交互路径。
- 清晰：顶部导航、侧导航、标题、字段、按钮层级明确。
- 高效：用户能快速定位当前页面、完成查看、编辑、认证、设置等任务。
- 规范：颜色、字体、间距、圆角、按钮、表格、弹窗遵循 UI 规范。
- 可编辑：交付 HTML 文件，结构清楚，方便继续修改。

## 2. 任务优先级

生成或修改页面时，按以下顺序判断：

1. 用户本次明确指令。
2. 用户截图中红框、标注或口头指出的问题。
3. 用户指定参考图。
4. 用户指定 UI 规范。
5. `AGENTS.md`。
6. `DH-skills.md` 或本文件。
7. 现有页面结构和交互。

如果用户明确说“其他不要动”，只允许修改指定区域。  
如果截图和规范不一致，优先按用户本次截图和说明执行。  
如果仍不明确，先问用户，不要自行大改。

## 3. 原型尺寸与文件

| 类型 | 建议尺寸 | 用途 |
| --- | --- | --- |
| Web 标准页面 | 1920 x 1080 | 主要高保真画面 |
| 主内容面板 | 1680 px 宽 | 内容不铺满超宽屏 |
| 中等宽屏适配 | 1440 / 1366 px | 检查导航和内容是否挤压 |
| 长页面 | 自适应高度 | 表格、协议、风险说明等 |
| 弹窗 | 依据内容，常用 720-800 px 宽 | 表单、认证、确认操作 |

文件命名：

```text
页面名称.html
页面名称-v2.html
页面名称-v3.html
```

规则：

- 新版本优先另存，不覆盖旧版本。
- 单文件 HTML 优先，除非用户要求拆分。
- HTML/CSS/JS 结构要清楚，便于继续编辑。
- 不删除旧文件。

## 4. 页面基础结构

推荐结构：

```text
页面底层
  顶部全局导航
  中间玻璃工作区
    左侧导航承载层
    右侧内容承载层
      标题栏
      页面内容
  浮动帮助入口
  弹窗/遮罩
```

三层视觉：

1. 页面底层：浅蓝到浅青背景，带柔和径向光斑，整体明亮、干净。
2. 中间工作区：半透明浅蓝玻璃层，浅白边框，轻微 blur，柔和蓝灰阴影。
3. 内容承载层：白色卡片、表格、列表或表单容器，浅蓝灰边框，轻阴影。

注意：

- 顶部导航通常是全宽浅色栏，不要强行做成圆角玻璃卡片。
- 侧导航和主内容区都属于内容承载层。
- 不要卡片套卡片过多，避免页面变碎。

## 5. 栅格与间距

| 项目 | 建议 |
| --- | --- |
| 页面主面板宽度 | 1680 px |
| 页面外边距 | 24 px 左右 |
| 工作区内边距 | 22-24 px |
| 左侧导航宽度 | 260-300 px |
| 左右栏间距 | 24 px |
| 内容区内边距 | 24-28 px |
| 普通详情行高 | 48-56 px |
| 头像/媒体详情行高 | 96-112 px |
| 表单字段间距 | 16-20 px |
| 表格单元格内边距 | 14-18 px |

原则：

- 同一页面同级模块间距保持一致。
- 详情行必须使用固定行结构。
- 头像、二维码、上传框等媒体行不能影响普通字段行距。
- 超宽屏内容不要铺满，优先保持主内容宽度稳定。

## 6. 色彩规范

| 用途 | 建议 | 说明 |
| --- | --- | --- |
| 页面背景 | 浅蓝到浅青渐变 | 页面底层 |
| 工作区背景 | 半透明浅蓝玻璃 | 中间工作区 |
| 内容卡片 | #FFFFFF / 极浅蓝灰 | 表单、表格、详情 |
| 主文字 | #142033 | 标题、重点内容 |
| 正文 | #56617C | 字段、说明、导航 |
| 弱文字 | #8D95A5 | 辅助说明、禁用 |
| 主交互 | #4486F5 到 #72C6DD | 主按钮、选中态 |
| 链接 | #4766FF | 查看、跳转 |
| 成功 | #2DB185 | 已认证、成功状态 |
| 警示 | #F1913D | 提示、预警 |
| 危险 | #FE4C4F | 删除、撤回、注销 |

使用原则：

- 主操作和选中态使用蓝色或蓝青色。
- 红、绿、橙只用于业务语义。
- 不用红色作为普通主按钮或普通导航选中。
- 不使用高饱和大面积装饰色。

## 7. 字体与排版

| 层级 | 字号/行高 | 字重 | 用途 |
| --- | --- | --- | --- |
| 品牌中文 | 18-22 / 22-28 | 800 | 顶部品牌 |
| 模块一级标题 | 22 / 28 | 800 | 我的账户、模块名 |
| 内容区标题 | 18 / 24 | 800 | 个人信息、企业认证 |
| 顶部导航 | 14 / 20 | 500-700 | 全局导航 |
| 正文/字段 | 14 / 20 | 500-600 | 字段名、字段值 |
| 子菜单/说明 | 13 / 18-20 | 400-500 | 辅助信息 |
| 徽标/弱提示 | 12 / 18 | 500-700 | 数量、状态 |

排版规则：

- 子页面标题不得大于所属模块主标题。
- 详情页不要使用过大字号。
- 表格、字段、按钮优先使用 14px。
- 用户名和顶部导航保持同级，不要过大。
- 下拉箭头、徽标、图标不能影响文字基线。

## 8. 顶部导航规范

顶部导航必须优先参考用户最新截图。

常见结构：

```text
Logo + 品牌文字 + 天玑标识
水平导航菜单
消息图标 + 红色徽标 + 头像 + 用户名 + 下拉箭头
```

样式要求：

- 顶栏全宽铺开，浅色背景。
- 顶栏高度约 72-80 px。
- 顶部导航文字使用 14px。
- 当前导航使用蓝色文字和蓝色下划线。
- 右侧用户区整体紧凑，不要像卡片。
- 消息图标使用线性图标，不使用大圆形按钮。
- 消息徽标贴近图标右上角。
- 头像使用 28-32 px。
- 用户名使用 14px / 20px。

关键约束：

- 不要把用户名和箭头写成 `用户0001⌄`。
- 用户名和箭头必须拆成独立元素。
- 箭头使用 CSS 或 SVG 绘制，并与文字垂直居中。
- 右侧区域不能过宽、过松或与参考图比例明显不一致。

## 9. 左侧导航规范

有侧导航时，应根据用户最新参考图确定样式。

推荐结构：

```text
侧导航承载层
  模块标题
  一级菜单
  可折叠一级菜单
    子菜单
```

视觉要求：

- 侧栏可使用浅蓝、浅青或半透明浅色背景。
- 侧栏应作为完整高度承载区域。
- 一级菜单默认可使用浅白底。
- 当前一级菜单可使用蓝到青渐变，白色文字。
- 子菜单选中态使用浅蓝底和蓝色文字。
- hover 态轻量，不要过重。

交互要求：

- 折叠菜单采用手风琴逻辑：一次最多展开一个一级菜单。
- 点击已展开菜单可收起。
- 多个菜单项指向同一页面时，只高亮当前点击项。
- 展开/折叠图标不能使用过小文本字符。
- 推荐使用圆形 chevron、CSS 箭头或 SVG。
- 图标需要默认、hover、展开状态。

## 10. 详情页规范

详情页常见结构：

```text
内容标题
详情列表
  字段名
  字段值
  操作
```

固定行结构：

```css
.profile-list {
  display: grid;
}

.profile-row {
  min-height: 56px;
  display: grid;
  grid-template-columns: 132px minmax(220px, 1fr) 180px;
  align-items: center;
  column-gap: 24px;
}

.profile-row.avatar-row {
  min-height: 104px;
}
```

规则：

- 普通字段行统一行高。
- 头像行、二维码行、上传行单独设置行高。
- 字段名列固定宽度。
- 字段值列自适应。
- 操作列固定宽度。
- 字段名、字段值和按钮垂直居中。
- 不使用自由 `grid gap` 混排媒体行和普通行。

## 11. 按钮规范

| 类型 | 用途 | 样式 |
| --- | --- | --- |
| 主按钮 | 保存、提交、确认 | 蓝青渐变 |
| 次按钮 | 编辑、修改、更新认证 | 蓝色描边 |
| 文本按钮 | 查看、链接跳转 | 蓝色文字 |
| 风险按钮 | 删除、撤回、注销 | 红色语义 |

按钮规则：

- 常规高度：32 px。
- 重要按钮高度：40 px。
- 圆角：999 px。
- 字号：14 px。
- 字重：600。
- 同类按钮必须统一。
- 禁用态要明显，但不能消失。

## 12. 表单与弹窗规范

表单控件：

- 高度：40 px。
- 字号：14 px。
- 背景：浅色输入框背景。
- 边框：浅蓝灰。
- focus：蓝色描边或浅蓝外发光。
- disabled：低饱和灰蓝背景。

弹窗结构：

```text
遮罩
弹窗容器
  标题栏
  关闭按钮
  表单内容
  操作按钮
```

弹窗规则：

- 弹窗标题使用 18px / 24px。
- 关闭图标要清晰，不要过大。
- 表单字段间距保持稳定。
- 底部操作按钮居中或按原页面逻辑摆放。

## 13. 表格规范

表格适用于协议清单、业务列表、高密度数据。

要求：

- 表头使用极浅蓝灰底。
- 表头和内容优先 14px。
- 行高稳定。
- 行 hover 使用浅蓝透明底。
- 操作列固定宽度。
- 查看类操作使用蓝色文本按钮。
- 撤回、删除类操作使用红色文本按钮。
- 表格不要使用厚重深色网格。

## 14. 状态与交互规范

页面至少考虑：

- 菜单：默认、hover、active、展开、收起。
- 按钮：默认、hover、disabled。
- 输入框：默认、hover、focus、disabled。
- 开关：开启、关闭、禁用。
- Tab：默认、hover、active。
- 标签：默认、hover、selected。
- 表格行：默认、hover。
- 弹窗：打开、关闭。

HTML 原型应实现：

- 左侧菜单切换页面。
- 折叠菜单手风琴展开。
- 弹窗打开/关闭。
- Tab 切换。
- 标签选中/取消。
- 开关切换。
- 勾选后启用按钮。

## 15. 局部修改规范

用户要求局部修改时：

- 只修改指定区域。
- 不顺手重构其他区域。
- 不调整无关颜色、布局、交互。
- 最终说明哪些区域被修改，哪些区域保持未动。

示例：

```text
用户说：只改顶部导航，其他不要动。
执行：只改 topbar 相关 CSS 和 HTML，不动侧栏、内容区、JS 逻辑。
```

## 16. 参考截图使用规范

使用截图时必须关注：

- 是否为最新参考图。
- 用户是否圈选具体区域。
- 参考的是整体风格，还是某个局部组件。
- 参考图里的尺寸、间距、字号、颜色、图标是否与当前页面冲突。

注意：

- 不要只说“参考了”，要真正按截图修比例。
- 顶部导航、右侧用户区、左侧导航这类区域，参考图优先级很高。
- 如果用户指出“不像”，应继续贴近参考图，而不是泛泛解释。

## 17. 生成前检查

生成或修改前检查：

- 是否读取 `AGENTS.md`。
- 是否读取本文件或 `DH-skills.md`。
- 是否读取用户指定 UI 规范。
- 是否查看用户指定截图。
- 是否确认输出文件名。
- 是否确认是新建文件还是修改已有文件。
- 是否确认用户有没有说“其他不要动”。

## 18. 交付检查清单

交付前逐项检查：

- 顶部导航是否和最新参考图一致。
- 顶部右侧用户区是否紧凑，头像、徽标、箭头是否对齐。
- 左侧导航是否符合参考图。
- 左侧折叠菜单是否一次只展开一个。
- 左侧展开图标是否清晰。
- 字号层级是否合理。
- 子页面标题是否没有大于模块标题。
- 详情字段行距是否均匀。
- 头像行是否没有影响普通字段行距。
- 按钮样式是否统一。
- 表格行高、表头、操作列是否稳定。
- 弹窗是否没有溢出。
- 页面是否没有文字重叠、遮挡、溢出。
- 用户要求局部修改时，是否没有改动无关区域。

## 19. 交付说明

交付时说明：

- 创建或修改了哪些文件。
- 修改了哪些区域。
- 哪些区域保持未动。
- 参考了哪些规范或截图。
- 是否运行了检查命令。
- 如果仍有风险或待确认点，明确列出。
